# TECHNICAL DESIGN ERD
<img width="1539" height="1368" alt="Entity Relationship Diagram" src="https://github.com/user-attachments/assets/da2a13ab-feed-4c57-a82a-50fb2f191b47" />
1. User
- Stores basic user info (UserId, UserName, Password).
- One user can have many roles.

2. Role
- Contains the list of system roles (RoleId, RoleName).
- One role can be assigned to many users.

3. UserRole
- Acts as a bridge table between User and Role.
- Defines which user has which role.
- Also has its own primary key: UserRoleId.

4. MasterPage
- Stores information about system pages a role can access.
- Linked to UserRole â†’ meaning different roles can access different pages.
